Issues from full-game audit (2026-06-11). Categories: ARCH = design-philosophy
violation, PERF = optimization, LEAK = memory/resource leak, BUG = correctness,
UX = player-facing experience, DEAD = dead code / undeleted old path.

--------------------------------------------------------------------------------
P0 — DESIGN-DOC VIOLATIONS / BIGGEST WINS
--------------------------------------------------------------------------------

[003][ARCH] rts-sim-wasm/src/lib.rs is a single 38,280-line file. Project rule:
split cohesive behavior into small modules. Split into ~14 modules along the
existing section seams (init/panic-hook; damage+economy ~387-2054; body pool
~3181-4595; sphere-sphere ~4398-4600; sphere-cuboid ~4596-4960; quaternion/pose
~4962-6190; projectile ~6901-7627; turret springs ~7638-8849; terrain+A*
~8851-11993; spatial grid ~11995-14524; combat targeting ~14549-25659;
reflection/submunition ~25660-26927; snapshot diff ~26933-33433; snapshot
envelope ~33433-end). Pure code motion, no behavior change; keep wasm-bindgen
exports stable; rebuild + run contract tests after.

[004][BUG] Rust panic risks on data that can arrive from JS/network in tick
paths: lib.rs:4762 indexes engine statics via .expect() on a handle;
lib.rs:3076 and 7239 use .try_into().unwrap() on input arrays
(step_unit_motion, solve_kinematic_intercept). A malformed command/buffer can
panic the authoritative sim. Fix: bounds-check and return sentinel (-1 / early
return) instead of panicking; keep debug_asserts for dev.

--------------------------------------------------------------------------------
P1 — LEAKS / PERF / UX
--------------------------------------------------------------------------------

[010][PERF] Snapshot encode path allocates per entity per snapshot:
stateSerializerEntities.ts builds nested turret/action/waypoint DTOs fresh each
snapshot; snapshotEntityWirePack.ts pools on DECODE but encode allocates fresh
row objects (84-97). At 10 snapshots/s x 1000 entities this is constant churn.
Fix: mirror the decode-side pooling on the encode side (reusable row/DTO pools,
length=0 array reuse).

[011][ARCH] Two live entity wire-encode pipelines: Rust V6
(snapshotRustWireEncoder.ts -> wasm) and the TS fallback
(stateSerializerEntities/snapshotEntityWirePack + msgpack). Delete-The-Old-Path
requires one. Fix: once Rust path is default-on and byte-stable (compare mode
exists for this), delete the TS entity encode path and keep TS decode only if
still needed; also confirm RUST_SNAPSHOT_WIRE_COMPARE default is OFF so the
duplicate full JS pack+msgpack at snapshotWireCodec.ts:171-173 never runs
unless explicitly enabled.

[012][PERF] NetworkSendBudget noncritical coalesce queue (512KB coalesce /
256KB flush, NetworkSendBudget.ts:4-7) has no hard cap — under sustained
backpressure pending noncritical messages can accumulate. Fix: cap the pending
queue (drop-oldest noncritical) so a stalled peer can never grow memory
unboundedly; count drops in net stats.

[013][PERF] TS terrain sampling allocates per query: terrainSurface.ts
projectHorizontalOntoSlope/applySurfaceTilt (~61-96) return fresh {x,y,z};
terrainTileMap.ts getTerrainMeshSample (~336-428) returns a fresh sample object
per call. These run per-unit-per-frame (render rigs) and per-tick. Fix: out-
param/scratch-object pattern like getSurfaceNormal already uses; audit callers.

[014][ARCH] Terrain height/normal sampling exists in BOTH TS
(src/game/sim/terrain/terrainTileMap.ts) and Rust (terrain mesh kernels in
lib.rs). Decide ownership: route sim-side queries through Rust; renderer may
keep a read-only client sampler but must share the same data, not a parallel
implementation. Document which is authoritative and delete/reduce the loser
per Delete The Old Path. (Verify exact duplication before cutting.)

[015][DEAD] src/game/sim/Pathfinder.ts keeps a module-scope building-mask cache
(~80-96) though the comment says mask+A*+LOS smoothing all run inside WASM.
Verify the TS cache is still consumed; if it is dead scaffolding from the
pre-WASM path, delete it (Delete The Old Path).

[018][ARCH] Optional chaining / optional fields in per-tick sim hot paths
(Explicit Absence rule): src/types/sim.ts:274-282 staticShield* fields are
`?:`; staticShield.ts:34,169-181 reads entity.body?./unit?. per tick;
projectileSystem.ts:526 weapon.shield?.deployedPose in the fire loop;
aimSolver.ts:205,538 options?./combat?.; targetingInputStamping.ts:706
entity.combat?.trajectoryMode per entity per tick. Fix: convert these specific
fields to `T | null` (initialized) or hoist explicit null guards out of the
loops; leave third-party boundaries alone.

[019][UX] Right-drag attack-line / formation-line flow is referenced by UI text
but discoverability is unverified: SelectionPanel "Atk Line" button sets a
waypoint mode whose right-drag activation is never explained in-UI, and the
"Assume" button title claims "Alt+Right click also works" (SelectionPanel.vue
~723-729) — verify Input3DManager actually implements Alt+RightClick; fix the
binding or fix the title. Also Wait button title promises Ctrl/Cmd+Shift
"insert next" (~822) — verify queueModeFromEvent distinguishes that; align
behavior/title.

P2 — SMALLER FIXES / HARDENING / CLEANUP
--------------------------------------------------------------------------------

[033][PERF] Rust micro-allocs: spatial-hash cell item Vec allocates on first
touch within a tick (lib.rs:~4436) and woke buffer resizes per call (~4452).
Pre-size at pool init / reuse cells across ticks.

[034][ARCH] Rust: spatial_query_occupied_cells_debug (lib.rs:~14490) iterates
HashMap order — debug-only, but sort keys so its output is stable for replay
diffing.

[035][ARCH] WASM boundary chatter: ~342 exported functions, several called
per-entity/per-frame from JS (e.g. wind_sample_state,
build_target_horizontal_distance). Identify the per-frame offenders and batch
them (one call over typed arrays) — measure first, then batch the top few.

[037][UX] Minimap UX papercuts batch (SelectionPanel.vue unless noted):
- select-only buttons read like counts ("Tower 2") — relabel "(2 towers)"
  (~696).
- factory unit buttons lack cost on the button face while build buttons show
  it (~1297) — add cost label.
- auto-group "A" badge is ~7px and easy to miss (~515-530) — add legend or
  tooltip.
- "Ground" (attack-ground) toggle title is ambiguous (~777-783) — clarify.
- client control bar HUD toggles are single letters (UNIT/TOWER/...) — expand
  labels where space allows (GameCanvasClientControlBar.vue).


[039][ARCH] Factory preset localStorage sync has a cross-tab race
(SelectionPanel.vue ~364-367). Low stakes: add a version field and
last-write-wins on the whole slot array.

[040][UX] Verify an explicit attack-move binding exists and is discoverable
(BAR ships one); commandHotkeys.ts shows fight/attack but no attack-move
entry. Add binding or document that right-click-on-ground = attack-move for
fight mode.

[041][PERF] isFarFromWater does 8 radial water samples per call
(terrainSurface.ts ~132-148). Verify call sites/cadence; precompute a water
mask if it is on a hot path.

[042][PERF] terrainTriangleSampleFromGlobalMesh iterates every triangle in the
cell per sample (terrainTileMap.ts ~331). Cache last-hit triangle per querier
(units rarely change triangle between frames).

turrets through the Rust solver; delete the TS solver path.

[051][ARCH] targetingInputStamping.ts (~957 lines) is the TS->Rust stamping
shim for the targeting FSM. Once [050] lands and targeting state lives fully
in Rust, eliminate the per-tick stamp/readback and the shim.

[052][ARCH] damageHelpers.ts keeps TS damage-context/knockback math parallel to
Rust segment/sphere kernels. Unify damage resolution in Rust; TS receives
events only.
