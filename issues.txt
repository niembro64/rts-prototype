MAP-DIMENSION HELPER DUPLICATION (NEW, found 2026-05-05)

1. Two different "force land-cell count to odd" rules across the codebase.
   References:
   - src/game/landGrid.ts:nearestOddLandCellCount   (rounds to NEAREST odd)
   - src/config.ts:normalizeMapLandCells            (rounds UP to next odd)

   Same goal (maps need exactly one central land/mana cell, which requires
   odd cell counts), different policy. nearestOddLandCellCount feeds the
   lobby axis-options generator (mapSizeConfig); normalizeMapLandCells is
   only used internally by config.landCellMapSpan and has no external
   callers. Either delete normalizeMapLandCells (it's effectively dead —
   landCellMapSpan via mapSizeFromLandCells could route through landGrid
   instead) or document the asymmetry so future readers know which rule
   applies where.

   Fix direction: drop the unused config-side normalizer once
   mapSizeFromLandCells is migrated, OR alias normalizeMapLandCells to the
   landGrid version so there's a single source of truth for the rounding
   policy.
