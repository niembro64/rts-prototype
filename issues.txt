[32] Optimization - reduce snapshot materialization cost
    Preserve deterministic-lockstep truth and current client reconciliation
    semantics. The presentation path has a 30 Hz publication opportunity from
    the fixed-step loop, rich scope/full-detail snapshots at 5 Hz, sparse
    entity-motion delta opportunities at 10 Hz, and projectile delta
    opportunities up to 30 Hz. Do not force empty heartbeat snapshots just to
    make the total SPS display read 30; non-empty entity/projectile deltas are
    event-driven. The work is to reduce the cost of building, serializing,
    applying, and materializing each meaningful snapshot. Use the existing
    received/applied cadence, bytes, materialization row/stage, and
    client-apply counters to identify the dominant stage, then move high-count
    snapshot materialization away from JS object/DTO churn and toward typed
    dirty slot/range reads from the canonical slabs. 
    
[24] Optimization - port Simulation.updateUnits hot action loop to native batches
    Preserve all command, action, path, arrival, stuck, guard, fight, patrol,
    attack, build, repair, reclaim, capture, resurrect, load, and unload
    semantics. Replace the branch-heavy per-unit TypeScript loop with
    slot-indexed Rust/WASM state machines and batch systems. Verify identical
    command-frame outputs, unit positions, action transitions, and checksums on
    golden replays.

[25] Optimization - remove UnitForceSystem JS pack/scatter from the tick path
    Preserve the existing force formulas and resulting movement. UnitForceSystem
    should stop gathering active IDs into JS arrays, packing Float64Array rows,
    sampling support/water inputs in TypeScript, and scattering results back to
    Entity objects. Move the required inputs/results into native slabs so the
    force batch reads and writes canonical state directly.

[26] Optimization - make PhysicsEngine3D sync slot-native and dirty-range based
    Preserve current physics behavior and collision/contact outcomes. Dynamic
    body integration, contact results, support-surface state, sleep/wake state,
    and dirty flags should update canonical slots directly. Avoid per-touched
    body JS entity writes on every physics step; expose bounded dirty ranges or
    slot lists for systems that need presentation updates.

[27] Optimization - make spatial queries fully native and shared across systems
    Preserve exact query semantics. SpatialGrid should not rely on JS Maps,
    entityBySlot references, JS result arrays, or per-entity wrapper resolution
    in hot paths. Sim, combat, projectiles, visibility, snapshots, and render
    scope should consume shared slot-native broadphase results.

[28] Optimization - move visibility/fog/radar filtering to native aggregate slabs
    Preserve current visibility gameplay exactly. Promote visibility/fog/radar
    from combat-targeting-owned coverage masks into a first-class native
    aggregate slab with LOS-aware full-sight output, scan-pulse integration,
    fog/minimap cell state, recipient/team dirty ranges, and direct consumers
    for render scope. Keep snapshot/minimap slot-mask parity tests in place
    until the remaining JS LOS/source fallback paths are removed.

[30] Optimization - flatten ClientViewState and prediction into slot-indexed typed state
    Preserve client prediction behavior and reconciliation results. Replace
    hot-path client Map<EntityId, Entity>, serverTargets maps, prediction sets,
    dirty sets, and render packet construction from JS objects with typed
    slot-indexed state. Render preparation should read stable presentation
    slots and dirty ranges instead of rebuilding object-derived packets each
    frame.

[31] Optimization - render high-count units from stable GPU instance buffers
    Preserve current visuals unless a separate gameplay/design issue approves a
    visual change. Reduce per-unit Three.js Object3D/EntityMesh orchestration in
    Render3DEntities. Units, turrets, chassis, shields, health/selection/range
    overlays, and far LOD proxies should update stable GPU instance buffers via
    dirty ranges and shader-side expansion/culling. Audit high-count primitive
    geometry usage through the existing PrimitiveGeometryQuality3D tier config
    before changing visuals: sphere/cylinder/cone segment reductions are valid
    only when triangle/GPU/render-submit metrics show material pressure, and
    should not be treated as a fix for SPS or snapshot materialization.
