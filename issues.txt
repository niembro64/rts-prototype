SNAPSHOT WIRE SIZE PLAN (SNAP-WIRE-01 family)

==============================================================================

------------------------------------------------------------------------------
SNAP-WIRE-01 - parent: snapshot payloads exceed the current per-client
               internet budget at small battle sizes.
------------------------------------------------------------------------------

Evidence:
- The latest 2026-05-22 fixed-point capture in
  docs/dp-02-snapshot-wire-instrumentation.md shows the roughly
  200-unit DIFFSNAP stream back inside budget: 52,554 B avg and
  114,475 B hi, against the 64 KiB avg / 128 KiB hi targets.
- The 2026-05-22 audio packed-row capture in the same doc compares the
  same snapshots through the previous Rust object-audio encoding and
  the new packed-audio wire path. DIFFSNAP avg moved from 63,633 B to
  50,212 B, while audioEvents moved from 16,136 B avg / 31,942 B hi to
  2,715 B avg / 5,186 B hi. Packed decode had zero event-count
  mismatches across 103 captured snapshots.
- The 2026-05-22 minimap/projectile packed-row capture in the same doc
  compares the previous Rust-backed audio-packed / object-minimap /
  object-projectile path against the new packed row wire path.
  DIFFSNAP avg moved from 58,922 B to 49,711 B, and hi moved from
  122,910 B to 98,067 B across 180 captured delta snapshots. The
  minimapEntities section moved from 4,343 B avg to 1,132 B avg, and
  projectiles moved from 9,397 B avg to 3,397 B avg. Packed decode had
  zero minimap/projectile count mismatches across 181 captured
  snapshots.
- The 2026-05-22 entity packed-row code-path lands per-entity flag/row
  packing for the `entities` top-level key. A synthetic 170-entity
  delta-shaped sample (100 pos+vel-only units, 50 keyframe buildings,
  20 hover units with orientation + angular velocity) measured raw
  msgpack at 17,148 B vs packed msgpack at 4,194 B (75.5% reduction)
  via the offline parity test. A 5-entity representative sample
  measured raw at 822 B vs packed at 198 B (75.9% reduction). Live
  2026-05-22 PLAYER CLIENT capture against the same 5-player demo /
  cap 243 / 32 SPS harness measured DIFFSNAP at 13,664 B avg /
  27,054 B hi across 445 deltas, and FULLSNAP bootstrap at 961,963 B.
  The previous audio + minimap + projectile packed-row capture measured
  49,711 B DIFFSNAP avg / 98,067 B hi and 1,069,273 B FULLSNAP
  bootstrap.
- The 2026-05-22 static terrain/buildability packed-row capture attacks
  the remaining FULLSNAP static sections. FULLSNAP bootstrap moved from
  961,963 B to 291,265 B. The `buildability` section moved from
  561,920 B to 11,933 B via exact run-length triples over flag/level
  cell pairs, and the `terrain` section moved from 357,285 B to
  236,331 B via compact binary numeric slabs. Static decode smoke
  passed: buildability flags/levels and all terrain integer arrays
  round-tripped exactly; Float32 terrain coord/height drift stayed
  below 0.0005 world units. DIFFSNAP stayed effectively unchanged
  at 13,803 B avg / 28,072 B hi across 441 deltas.
- After audio, minimap, projectile, entity, terrain, and buildability
  packing, the cap-243 harness is well inside both the 64 KiB DS avg /
  128 KiB DS hi and the 1 MiB FS targets. The remaining proof is a
  fresh PLAYER CLIENT capture at the 1,000 / 5,000-unit rows against
  the same network budgets.

Why it matters:
Snapshot bandwidth is per remote client and burns host upstream first.
Even when the 200-unit delta stream is under target, every remote client
multiplies host upstream, FULLSNAP/static sections still need a targeted
budget pass, and the 1,000 / 5,000-unit rows still need to be proven
against the same network budgets.

Fix direction:
Do child items as separate, measurable PRs. The remaining priority is
to rerun the PLAYER CLIENT DS/FS capture matrix at the 1,000 / 5,000
unit rows and record the same DS SIZE / FS SIZE avg + hi numbers
against the documented network budgets.
