Performance audit (2026-05-08): areas where we can use better algorithms or
fewer passes over the same items. Each cluster includes the cost model and
the proposed remedy. Verify against current code before attacking — a few
related sites may have already shifted.

PERF-3. Selection caches rebuilt by full entity walk
  Sites:
    - src/game/scenes/RtsScene3D.ts:1439-1456 (rebuildSelectedEntityCachesIfNeeded)
    - src/game/scenes/RtsScene3D.ts:1421-1437 (preferUnitsOverBuildingsInSelection)
  Cost: When the dirty flag fires (every selectEntity/deselectEntity call),
    these scan ALL units and ALL buildings to find the ones with
    `selectable.selected === true`. Worst case during band-select drag, this
    refreshes per drag tick. O(N_total) per refresh where N_selected ≪ N_total.
  Proposed: Maintain a `Set<EntityId>` of currently-selected entities updated
    inline by selectEntity / deselectEntity in ClientViewState. Cache rebuild
    becomes O(N_selected) — iterate the set, resolve to entity, push into
    the typed cache.

PERF-4. `getSurfaceNormal` resampled across same-tick callers
  Sites:
    - src/game/sim/combat/ProjectileCollisionHandler.ts:702-708 (fired per
      cluster-munition splash; multiple projectiles colliding in the same
      tick at near-identical (x,y) each resample)
    - src/game/network/ClientProjectileStore.ts:236 (client-side projectile
      prediction)
    - src/game/render3d/SelectionOverlayRenderer3D.ts:185 (selection ring
      fallback when entity.unit?.surfaceNormal is missing)
    - src/game/sim/Pathfinder.ts:170 (slope-aware pathing — per pathfind
      call per candidate cell)
  Cost: `getSurfaceNormal` does a bilinear gradient over the heightmap. The
    sim and render fields `unit.surfaceNormal` already cache per-unit
    (good — keep). The remaining hotspots are projectile-burst splash
    (cluster munitions) and pathfinder cell-slope reads. ProjectileCollision
    case can collapse N samples per tick to 1 when several submunitions land
    in the same grid cell.
  Proposed: Either (a) a tiny LRU keyed by `floor(x/cellSize)|floor(y/cellSize)`
    that resets on tick boundary, or (b) explicitly hoist the normal lookup
    above the projectile-iteration loop in ProjectileCollisionHandler when
    splash fires multiple subs at the same impact cell. Pathfinder's per-cell
    sampling is already amortized over the whole pathfind so lower priority.

PERF-5. `unit.transform.rotation` cos/sin not flowing into the renderer for
  per-turret mirror panels
  Sites:
    - src/game/render3d/Render3DEntities.ts:1310-1318 (mirror panel
      orientation: 4 raw Math.cos/sin per mirror-bearing unit per frame)
  Cost: Sim already publishes `transform.rotCos / rotSin` and there's a
    centralized `getTransformCosSin` helper (used at targetingSystem.ts:65
    as the producer). The renderer should be a consumer; computing fresh
    cos/sin for the chassis rotation when the cached value is one prop-read
    away duplicates work the sim already paid for.
  Proposed: Replace the chassis term with `const { cos: cosMirrorRot,
    sin: sinMirrorRot } = getTransformCosSin(turretedEntity.transform);`.
    Pitch is per-turret and not cached, so leave that pair as-is. Only
    saves 2 trig ops per mirror-bearing unit per frame — small absolute
    gain, but it's a "do the consistent thing" cleanup that prevents the
    cache from rotting.
