    from step() at line 703 immediately after the force/integration pass —
    RULE: "Mass must matter" (philosophy line 716): "Ground contact is spring and

A25. [LOW] rts-sim-wasm/src/lib.rs compute_arrival_control_thrust (~681-808), the
    ARRIVAL_FLAG_FLYING branch —
    RULE: "Locomotion is arrival control, not waypoint seeking" (philosophy lines
    719-749): every locomotion profile computes force from BOTH position error AND
    velocity error; hovercraft and air "use the same high-level arrival controller;
    they differ only by mass, thrust, friction, damping, turn authority, contact
    model, and vertical constraints." —
    PROBLEM: for FLYING units the arrival controller takes a position-only branch (seeks
    the waypoint without the braking/velocity-error term ground and hover units use), so
    air units can overshoot/orbit — the exact failure the rule exists to prevent. Ground
    and hover paths correctly use position+velocity error with thrust saturation (audited
    CLEAN). —
    FIX: route flying units through the same velocity-aware arrival term (position error +
    velocity-error braking, saturated against thrust), differing only by the flying unit's
    physical limits, instead of a position-only seek branch.

A26. [MEDIUM] src/game/sim/blueprints/shots.json (lightRocket ~:66, fastRocket ~:89,

    RULE: "Gravity is universal; thrust answers it" (philosophy lines 751-777): "No

    inside the same homing/steering update." —
    PROBLEM: guided rockets set gravityForceMultiplier:0 — exactly the "flat trajectory
    flag that bypasses integration" the doc forbids. They fly level by switching gravity
    OFF rather than by adding a bounded counter-gravity thrust term in the homing update,

    grep of shots.json shows gravityForceMultiplier:0 on the rocket entries; non-guided
    shots correctly use 1.) —
    FIX: set gravityForceMultiplier back to 1 for guided rockets and have the homing
    update add a vertical counter-gravity thrust term (bounded by available thrust) so
    flight looks flat while gravity stays integrated; when guidance/thrust is lost the
    rocket sags ballistically. (Pairs with A27.)
