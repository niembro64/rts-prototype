BAR COMMAND AND UI PARITY GAPS

Baseline: budget_design_philosophy.html says player-facing behavior defaults to
Beyond-All-Reason unless explicitly overridden. BAR references for this backlog:
https://www.beyondallreason.info/commands-20 and
https://github.com/beyond-all-reason/Beyond-All-Reason/tree/master/luaui/configs/hotkeys.

Issue: the shared command-hotkey registry and in-game preset switcher only cover
commands that already exist in rts-prototype, with prototype, BAR-grid, and
BAR-legacy presets for that subset, runtime multi-key sequence dispatch, and
build-slot hotkeys covering the current nine-structure build roster.
Finish the BAR-compatible keymap and command UI layer by importing the complete
BAR grid/legacy bindings and adding command buttons for missing BAR commands so
visible hotkeys match the active keymap across the whole command surface.

Issue: metal-extractor upgrade workflows are missing. Implement T1 mex area
upgrade and single T1 -> T2 upgrade replacement workflows with construction
lifecycle, replacement rules, command authorization, hotkeys, and visible
previews/buttons.

Issue: queued-order insertion needs a client-facing commandinsert affordance
for choosing a specific `queueInsertIndex` instead of only append or
queue-front.

Issue: behavior/state toggles are incomplete. Implement missing BAR-equivalent
states such as cloak/want-cloak, gather wait, richer stance/fire states beyond
the current fire/hold-fire toggle, richer positioning/move states beyond the
current maneuver/hold-position toggle, and any related stateful command buttons
in simulation, snapshots, command authorization, selection UI, hotkeys, and
visible state indicators.

Issue: special abilities are incomplete. Add BAR Capture, Load Transport,
Unload Transport, Resurrect, Resurrect Area, manual launch, and any
unit-specific special buttons beyond commander DGun only after the underlying
authoritative simulation, snapshots, client feedback, and physical
presentation exist.

Issue: BAR gameplay functionality tied to missing commands is absent. Implement
the systems needed by the missing command surface: transports, capture,
resurrection/wreck fields, cloak/detection, unit stances, manual-launch
weapons, mex upgrade workflows, replays where applicable, and complete
faction/unit/factory/building rosters.

Issue: BAR parity must be extended through the 3D/physics-first design, not
implemented as flat 2D shortcuts. Transport/load/unload, resurrection,
capture, and manual launch/special-fire abilities beyond commander DGun need
physical presentation and authoritative physics interactions.
