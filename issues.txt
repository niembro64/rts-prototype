TERRAIN / MAP GENERATION OPTIMIZATION AUDIT

1. Build placement diagnostics duplicate terrain footprint checks.
   Reference: src/game/sim/buildPlacementValidation.ts:101
   Reference: src/game/sim/buildPlacementValidation.ts:128
   Reference: src/game/sim/buildPlacementValidation.ts:139

   The code checks the whole building footprint, then checks every fine
   placement cell with isBuildableTerrainFootprint, then separately asks for
   plateau levels. That repeats terrain mesh sampling heavily while moving a
   building ghost around.

   Fix direction: create one terrain-footprint diagnostic helper that walks
   the candidate cells once and returns: blocking terrain cells, common
   plateau level, water/steepness failures, and per-cell debug colors.

2. Public terrain APIs still accept arbitrary cellSize even though LAND_CELL_SIZE
   is supposed to be canonical.
   References:
   - src/game/sim/terrain/terrainTileMap.ts:23
   - src/game/sim/terrain/terrainTileMap.ts:184
   - src/game/sim/terrain/terrainBuildability.ts:26
   - src/game/landGrid.ts:27

   The project direction is that LAND_CELL_SIZE, spatial grouping, capture/mana
   tiles, and terrain cells are one canonical grid. Terrain APIs still accept
   arbitrary cell sizes and silently normalize them, which keeps a path open
   for frontend/backend drift.

   Fix direction: assert canonical LAND_CELL_SIZE at API boundaries using
   assertCanonicalLandCellSize, or remove the optional cellSize parameter from
   terrain public APIs that should never vary.

3. Map-size option generation is isolated from land-grid validation.
   References:
   - src/mapSizeConfig.ts:31
   - src/mapSizeConfig.ts:41
   - src/game/landGrid.ts:23

   mapSizeConfig has its own odd-cell rounding and validation, while landGrid
   owns canonical cell-size normalization and grid metrics. These are related
   policies but currently split across two places.

   Fix direction: move odd-cell option generation and default validation into
   a shared map/grid config helper so map dimensions, central-cell guarantees,
   and LAND_CELL_SIZE grid math are enforced from one module.
